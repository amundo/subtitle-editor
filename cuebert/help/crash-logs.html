Crash and Log Help

If Cuebert crashes or behaves unexpectedly, send the app log and the newest macOS crash report. These files help identify what happened.

Find the Cuebert App Log

  1. Open Finder.
  2. Choose Go > Go to Folder....
  3. Paste this location and press Return:
~/Library/Logs/Cuebert
  1. Attach the newest version-specific log, such as cuebert-0.8.2.log.
  2. If there is no version-specific log, attach cuebert.log.
If the folder is empty, open Cuebert again, reproduce the problem if you can, then check the folder again.

Find the macOS Crash Report

  1. Open Finder.
  2. Choose Go > Go to Folder....
  3. Paste this location and press Return:
~/Library/Logs/DiagnosticReports
  1. Search for Cuebert.
  2. Sort by Date Modified.
  3. Attach the newest file that starts with Cuebert.

The crash report may end in .ips or .crash. Either one is useful.

Send the Files

Send these files together: 

cuebert-0.8.2.log
the newest Cuebert crash report

Include a short note about what you were doing just before the crash, and whether the crash happens every time or only sometimes.

If Cuebert Will Not Open

You can still use Finder to collect the same files. The log and crash report locations are outside the app, so they remain available after a crash.
